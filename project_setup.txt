# 🚀 Welcome to the Sentinel Monorepo!

Welcome to Team MEGATRON! You are about to deploy Sentinel—our spatial parametric insurance platform powered by event-driven AI agents. This guide will get your local environment running the entire Dockerized stack in under 5 minutes. Let's build!

## 🛠️ Prerequisites

Before you begin, ensure you have the following installed on your machine:
- **Docker Desktop**: For running the containerized microservices.
- **Git**: To clone and manage the repository.
- **Ollama (Optional)**: If you plan to run local LLMs instead of using Google Gemini.

---

## 🔐 Step 1: The Secrets Override (CRITICAL)

Sentinel relies on strict secrets isolation to protect our API quotas. 

1. Duplicate the template file:
   In the `Sentinel` directory, copy `.env.example` to a new file named `.env` in the same infrastructure folder.
   
2. Generate your API Keys:
   Get your personal Google Gemini API key and insert it into your new `.env` file (`GEMINI_API_KEY=your_personal_key_here`). Do the same for `NEXT_PUBLIC_MAPPLS_ACCESS_TOKEN`.
   
3. ⚠️ **WARNING: NEVER COMMIT YOUR `.env` FILE.** It is already listed in `.gitignore`, but double-check before pushing any commits!

---

## 🚀 Step 2: Ignition

It's time to bring the infrastructure online. Open your terminal in the `Sentinel` directory (where `docker-compose.yml` lives) and run the following command to build and start all microservices:

```
docker-compose up --build
```

## 🚨 Troubleshooting (The "Oh No" Section)

If you encounter issues during setup, check these common fixes:

- **Port conflicts (e.g., "Port 5432 is already in use")**: 
  You likely have a local Postgres instance running. You must kill the local Postgres service or disable it so Docker can bind to port 5432.
  
- **Volume permission issues for n8n**: 
  If n8n fails to start due to write access, ensure the `Sentinel/infrastructure/n8n/local_files` directory has the correct permissions for Docker to write to it.
